Nu Vibing Project Guide

Step-by-step reference for evolving this Expo React Native prototype, focused on design testing.

How to Add a New Screen

1

Create the file in screens/. Use ScreenTemplate from ../components and useTranslation from ../i18n.

import { ScreenTemplate } from '../components';
import { useTranslation } from '../i18n';

export default function MyNewScreen({ locale, onBack }) {
  const t = useTranslation(locale);
  return (
    <ScreenTemplate headerTitle="My Screen" onBack={onBack}>
      {/* your content */}
    </ScreenTemplate>
  );
}
2

Register the screen in config/screens.registry.ts with a unique name, transition preset, and description.

3

Add to a flow in config/flows.ts by adding a step entry referencing the screen name.

4

Wire into App.tsx by adding the navigation condition to render the new screen.

How to Add a New Language

1

Copy i18n/en.ts to a new file (e.g. i18n/fr.ts) and translate all strings.

2

Add the locale code to the Locale type in i18n/types.ts.

3

In i18n/index.ts: import the file, add to LANG_MAP, SUPPORTED_LOCALES, and LOCALE_FLAGS.

The StartScreen will automatically show a card for the new language.

How to Add a New Transition

1

Add a new preset object in transitions/presets.ts inside the PRESETS record.

2

Add the name to the TransitionPresetName union type in the same file.

3

Reference it in screen registry entries or flow steps.

How to Create a New Flow / Experiment

1

Add a new entry to the FLOWS object in config/flows.ts with a unique id, locale, and step sequence.

2

Change ACTIVE_FLOW_ID to the new flow's id.

3

Restart the app.

How to Add a Screen from Figma

1

Get the Figma link for your design.

2

In Cursor, paste the Figma link and ask the AI to implement the screen.

3

Place the generated file in screens/.

4

Follow "How to Add a New Screen" steps 2-4.

Tips

  • Original files are safe in backup/.
  • Use ScreenTemplate for consistent header, scroll, and safe-area handling.
  • Test on both iOS and Android via Expo Go.
  • To switch experiments, change ACTIVE_FLOW_ID in config/flows.ts.
  • All translation types are enforced by TypeScript — if you miss a key, you'll get an error.
